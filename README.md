### Backend:

# Платформа Достижений Студентов (Backend) 🏆

REST API сервис для учета портфолио, достижений и геймификации школьников и студентов. Построен на Django и Django REST Framework.

## 🛠 Технологии
* Python 3.x
* Django 6.x
* Django REST Framework (DRF)
* Авторизация: DRF Token Auth
* Документация: drf-spectacular (Swagger/OpenAPI)
* CORS Headers (для связи с фронтендом)

## 🚀 Быстрый старт (Локальная разработка)

1. **Клонируйте репозиторий:**
   ```bash
   git clone <ссылка_на_ваш_репозиторий>
   cd Platform_for_the_achievements_of_schoolchildren_and_students
   ```

2. **Создайте и активируйте виртуальное окружение:**
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate  # Для Mac/Linux
   # .venv\Scripts\activate   # Для Windows
   ```

3. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```
   *(Убедитесь, что у вас установлен django, djangorestframework, drf-spectacular, django-cors-headers, pillow)*

4. **Примените миграции базы данных:**
   ```bash
   python manage.py migrate
   ```

5. **Создайте суперпользователя (Админа):**
   ```bash
   python manage.py createsuperuser
   ```

6. **Запустите сервер:**
   ```bash
   python manage.py runserver
   ```

## 🎮 Настройка Геймификации (Обязательно после установки)

Система автоматически выдает бейджи при достижении определенных условий (сигналы). Чтобы это работало, нужно создать базовые бейджи в админке:
1. Зайдите в `http://127.0.0.1:8000/admin/`
2. Перейдите в раздел **Portfolio -> Бейджи**
3. Создайте 5 бейджей со строго следующими названиями (и загрузите для них иконки):
   * `Инноватор` (выдается за 3 хакатона)
   * `Марафонец` (выдается за 10 пройденных курсов)
   * `Командный игрок` (выдается за 5 командных проектов)
   * `Наставник` (выдается за 5 наставничеств)
   * `Первая победа` (выдается за 1 олимпиаду)

### 📝 Инструкция для Фронтенд-разработчика (Java/Web)


**Привет! Бэкенд Платформы Достижений готов к интеграции. Вот всё, что нужно знать для работы с API:**

**1. Базовый URL и Документация**
* Все запросы идут на: `http://127.0.0.1:8000` (пока работаем локально).
* Интерактивная схема всех методов (Swagger): **`http://127.0.0.1:8000/api/docs/`**. Там можно посмотреть все JSON-структуры и протестировать запросы.

**2. Авторизация (Токены)**
API закрыто от посторонних. Чтобы получить данные, нужен Token.
* **Шаг 1:** Отправь `POST` запрос на `/api/login/` с телом `{"username": "логин", "password": "пароль"}`.
* **Шаг 2:** В ответ придет JSON: `{"token": "твой_длинный_токен"}`.
* **Шаг 3:** Во все последующие защищенные запросы (к профилю, достижениям) добавляй HTTP-заголовок:
  `Authorization: Token твой_длинный_токен` *(Обрати внимание на слово Token и пробел)*.

**3. Структура профиля и Геймификация**
Когда ты запрашиваешь профиль пользователя (`GET /api/users/{id}/`), бэкенд отдает максимально развернутую структуру, чтобы тебе не пришлось делать лишние запросы.
* `avatar_details` — содержит готовую абсолютную ссылку на картинку аватара (поле `image`).
* `earned_badges` — массив полученных достижений. Внутри каждого элемента есть объект `badge` с названием, описанием и ссылкой на `icon`. Бэкенд сам высчитывает и выдает их при изменении статусов достижений куратором.
* `url` — готовая гиперссылка на этот конкретный ресурс (HATEOAS).

**4. Работа с картинками**
Ссылки на аватарки и иконки бейджей приходят в виде абсолютных путей (например, `http://127.0.0.1:8000/media/avatars/robot.png`). Ты можешь сразу вставлять их в UI компоненты.

**5. CORS**
CORS на бэкенде настроен. В данный момент разрешены запросы с `http://localhost:3000`, `127.0.0.1:3000` и `8080`. Если у твоего клиента другой порт (например, Android-эмулятор),то нужно добавить его в белый список в `settings.py`.
